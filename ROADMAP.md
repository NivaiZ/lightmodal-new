# LightModal — Roadmap (что довести до “идеала”)

Этот документ — список улучшений и возможных фич для развития текущей реализации LightModal до уровня “полноценной модалки для контента + медиапросмотра”, сохраняя при этом простоту интеграции.

> Важно: **не планируется** добавлять полноценный “carousel engine” или делать отдельную систему слайдов как в специализированных галереях.
> Галерея остаётся **лёгкой**: базовые prev/next + опциональный свайп без сложной кинематики.

## Принципы

- **Безопасный дефолт**: ничего “магического” без явного включения опциями.
- **Нулевая зависимость** по умолчанию (всё расширяемое — через плагины/адаптеры).
- **Стабильный API**: минимум breaking changes, версионирование и changelog.
- **Доступность**: клавиатура, фокус, screen readers и prefers-* должны быть first-class.
- **Производительность**: lazy-loading, отмена запросов, минимум reflow.

## 1) Качество ядра (маленькие, но важные)

### 1.1 Единая модель опций и документация
- Синхронизировать `defaults` ↔ README ↔ data-атрибуты.
- Явно описать типы опций (boolean/number/string) и правила парсинга `data-lm-*`.
- Добавить `LightModal.version` в README + changelog.

### 1.2 Более строгая обработка ошибок
- Единый формат ошибок: `code`, `message`, `details`.
- Хуки/ивенты для ошибок: `error` / `ajaxError` / `loadError`.
- `errorTpl` с поддержкой безопасной вставки текста (без риска HTML‑инъекций).

### 1.3 Управление жизненным циклом
- Явные состояния: `Init → Opening → Ready → Closing → Destroyed`.
- Гарантии для `close()` (idempotent, всегда resolve).
- Опция `destroyOnClose` (по умолчанию true), чтобы можно было оставлять DOM для reuse.

## 2) Расширяемость: плагинная система (ключевой шаг)

### 2.1 API плагинов (минимальный, но мощный)
- `plugins: [pluginA, pluginB]` или `{ name: plugin }`.
- Плагин получает `{ instance, options }` и может:
  - подписываться на события (`instance.on(...)`)
  - добавлять DOM (toolbar, индикаторы, доп. UI)
  - возвращать `cleanup()` для `destroy()`.

Пример интерфейса (концепт):
- `plugin.setup(instance) → cleanup`
- `plugin.defaults` для merge с опциями

### 2.2 “Адаптеры окружения”
- Scroll adapter (уже есть Lenis) → расширить до `scrollAdapter` опции:
  - `getScrollY()`, `lock()`, `unlock()`, `scrollTo(y)`
- Focus adapter (для кастомных фокус-менеджеров)

---

## 3) UI‑слой: единый тулбар и элементы управления

### 3.1 Toolbar (опционально)
- Кнопки: close, prev/next, fullscreen, download, zoom, autoplay.
- Позиции: top/bottom/inside.
- Полная кастомизация шаблонов + классов.

### 3.2 Idle‑режим (расширить)
- Опция `idle: false | number` уже есть — добавить:
  - `idleIgnoreSelectors` (например, формы/инпуты)
  - “появление UI при движении мыши/тапе”

### 3.3 Счетчик / подпись
- Счетчик перенести в тулбар (если он включён).
- Caption: поддержка HTML (опционально и безопасно), перенос строк, max-height, scroll.

---

## 4) Контент и рендеринг: items, загрузка, кеш

### 4.1 Унифицированная модель “item”
- Поля: `src`, `type`, `thumb`, `caption`, `alt`, `width/height`, `fetchOptions`, `html`, `data`.
- Возможность передать `html: Element|string` напрямую (без `src`).

### 4.2 Preload (мягкий, без карусели)
- `preload: 0|1|2` — подгружать “соседние” элементы галереи (prev/next), без отдельного carousel-слоя.
- Отмена preload при смене текущего элемента.

### 4.3 Кеширование
- `cache: true|false` + LRU на N элементов.
- Для AJAX/JSON: кеш по URL+опции (опционально).

---

## 5) Жесты и интерактив для медиаконтента

### 5.1 Zoom/Pan (для изображений)
- Двойной клик/тап: zoom in/out.
- Drag pan при зуме.
- Pinch‑to‑zoom (touch).
- Ограничение границ, инерция, reset.

### 5.2 Swipe‑навигация (без “карусельной” логики)
- Оставить свайп **как удобный жест**, но без сложной “карусельной” физики:
  - threshold
  - отключение swipe при активном zoom
  - аккуратное различение “скролл vs свайп”

### 5.3 Fullscreen
- Опция `fullscreen: true|false` + кнопка.
- Корректное поведение `Escape`/фокуса.

---

## 6) Клавиатура и доступность (A11y)

### 6.1 Маппинг клавиш
- Опция `keyboard` как объект:
  - `{ Escape: 'close', ArrowLeft: 'prev', ArrowRight: 'next', ... }`
- Возможность отключить/переопределить на уровне instance.

### 6.2 ARIA и семантика
- `aria-label` для контейнера (опция).
- Правильные роли для toolbar/buttons.
- Объявление изменений контента (aria-live) для caption/counter (опционально).

### 6.3 Focus management
- Улучшить trapFocus:
  - учитывать `inert`/`aria-hidden`
  - корректно работать при динамических изменениях DOM
- Опция `initialFocus` (селектор/элемент/false).

---

## 7) AJAX/JSON: мощнее, безопаснее, удобнее

### 7.1 Режимы рендера
- `ajaxSelector` уже есть — добавить:
  - `ajaxTransform(html|doc)` для нормализации ответа
  - `ajaxError(err)` для кастомного UI

### 7.2 Санация
- Текущая санация — базовая. Добавить:
  - опция `sanitize: true|false|fn`
  - возможность подключить внешний санитайзер через плагин

### 7.3 Прогресс и отмена
- Показ прогресса (если есть `Content-Length`).
- Более явные таймауты: `ajaxTimeout`, `loadTimeout`.

---

## 8) Темизация и стили (CSS API)

### 8.1 Стабильный набор CSS‑переменных
- Зафиксировать публичные `--lm-*` и не ломать их между минорными версиями.
- Вынести “тёмную/светлую” тему в отдельные пресеты (через `[data-theme]`).

### 8.2 Layout‑варианты
- “media” режим: без padding, строго по aspect-ratio.
- “html” режим: padding + scroll внутри.
- Авто‑переключение по типу контента.

---

## 9) Интеграции (опционально)

- Smooth scroll: расширить кроме Lenis (через адаптер).
- Аналитика: хуки `onOpen/onClose` с метаданными (src/type/gallery).
- Router: закрытие по navigation, поддержка hash‑state (опционально).

---

## 10) DX: разработка, тесты, релизы

### 10.1 Сборка (опционально)
- Скрипт build для минификации JS/CSS.
- Source maps для dev.

### 10.2 Тесты
- Минимальные e2e smoke‑тесты:
  - open/close, focus restore, scroll lock, gallery nav, ajax/json.

### 10.3 Примерная матрица совместимости
- Chrome/Firefox/Safari/Edge
- iOS Safari: scroll/viewport, address bar, pinch gestures.

---

## Предлагаемый порядок внедрения

1) **Плагинная система + keyboard mapping** (даёт масштабирование без усложнения ядра)  
2) **Toolbar + fullscreen + preload** (ощущение “полного продукта”)  
3) **Zoom/Pan** (самая сложная часть, лучше делать после стабилизации API)  
4) **AJAX upgrades + sanitize hooks**  
5) **Тесты/релизный процесс**
